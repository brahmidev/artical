Build a Simple JavaScript Image Slider From Scratch
(No Libraries Required)

Building an image slider from scratch might seem like a task that requires jQuery or a dedicated plugin. However, with just HTML, CSS, and JavaScript, it’s possible to create a clean, functional slider without any external libraries.

Preview the Image Slider

Before diving into the code, here’s a quick preview of the image slider that we will be built in this tutorial.

Now, let's begin building the image slider from the ground up. We’ll start with the HTML structure, move on to the CSS for styling, and finish by adding JavaScript to bring the slider to life.

Let’s begin with the basic structure of our slider. This will give us the foundation to build on.

This HTML sets up the slider container. Inside it, the .slider div holds all the images, each with an alt attribute for accessibility. To ensure the first image is visible by default, we add a class active to it. Later, we'll use CSS and JavaScript to switch between slides.

Below the slider, there are two buttons: one with the class prev for going back, and one with the class next for moving forward.

Step 2: Styling the Slider with CSS

Now, let's move on to styling. I’ll break the CSS into sections so it’s easier to understand.

This CSS block styles the main container for our slider, centering it on the page, setting a fixed size, and adding some polished visual effects.

display: flex; align-items: center; justify-content: center; These properties center the content both vertically and horizontally inside the container.

margin: 0 auto; centers the container horizontally by setting the top and bottom margins to 0 and the left and right margins to auto, perfect for centering block-level elements like divs.

position: relative; This allows us to position elements (like buttons) absolutely inside the container later on.

max-width: 800px; max-height: 450px; overflow: hidden; These properties set a fixed size for the slider and ensure that any content extending beyond the container is hidden.

border-radius: 10px; box-shadow: 0 4px 10px rgba(0, 0, 0, 0.2); Used to give the slider rounded corners and a soft shadow, providing a modern, polished look.

For now, the fixed dimensions help maintain simplicity, but later you can make this more responsive using percentages or media queries.

.slider { display: flex; } This lines up all the slides horizontally.

.slide { min-width: 100%; display: none; } Each image (or slide) stretches to fill the full width of the container, but stays hidden by default.

.slide.active { display: block; } Only the active slide the one currently being shown gets displayed. This is the same class we added in the HTML to the first image.

For the button layout and alignment: position: absolute; top: 50%; transform: translateY(-50%); This places the buttons inside the container and centers them vertically .
background-color: rgba(0, 0, 0, 0.5); A semi-transparent black background keeps the buttons visible without covering too much of the image, and color: #fff makes the icons stand out.

.prev { left: 10px; } and .next { right: 10px; } - These position the "previous" button on the left side and the "next" button on the right.

With all the styling in place, our slider now looks great but it doesn’t do anything yet. Let’s jump into the JavaScript and make it fully functional by making the slides actually move when we click the navigation buttons.

Step 3: Making It Work with JavaScript

Time to add some magic, let’s make our slider fully functional. Using JavaScript, we’ll bring everything together and handle the interactive parts by:

  • Making the "next" and "previous" buttons work
  • Setting up automatic slide transitions every 20 seconds

Let’s jump in and see how this works!

1. Setup and Selecting Elements

First things first, we’ll grab the key elements we need and create a simple variable to keep track of which slide is showing. This sets the foundation for everything that follows.

JavaScript 

      
1let currentSlide = 0;

      
2const slides = document.querySelectorAll('.slide');

      
3const prevBtn = document.querySelector('.prev');

      
4const nextBtn = document.querySelector('.next');

let currentSlide = 0; This variable tracks the current slide. It’s initialized to 0, which means the first image will be shown initially.

const slides = document.querySelectorAll('.slide');,
const prevBtn = document.querySelector('.prev'); and const nextBtn = document.querySelector('.next'); We need to select all the slides and the navigation buttons for controlling the slider.

2. Displaying the Active Slide

Now, let's jump into the functions. We need a way to display the correct slide (image). This function will ensure the right image is shown as we move forward or backward.

JavaScript 

      
6function showSlide(index) {

      
7  slides.forEach((slide, i) => {

      
8    slide.classList.remove('active');

      
9    if (i === index) slide.classList.add('active');

      
10  });

      
11}

slides.forEach((slide, i) => { ... }); This loops through all the slides, giving us access to each one individually. The i represents the index of each slide in the slides NodeList.

slide.classList.remove('active'); Before displaying a new slide, we remove the active class from all slides.

if (i === index) slide.classList.add('active'); Then we check if the current slide’s index matches the index passed to the showSlide function. If it does, we add the active class, which makes that slide visible.

3. Navigating Between Slides

Next, we need to handle the navigation between slides: moving forward and backward when the user clicks the buttons. We'll use two functions.

JavaScript 

      
13function nextSlide() {

      
14  currentSlide = (currentSlide + 1) % slides.length;

      
15  showSlide(currentSlide);

      
16}

      
17

      
18function prevSlide() {

      
19  currentSlide = (currentSlide - 1 + slides.length) % slides.length;

      
20  showSlide(currentSlide);

      
21}

nextSlide() This function advances to the next slide by increasing currentSlide by 1. The expression (currentSlide + 1) % slides.length ensures that when we reach the last slide, it loops back to the first one, creating a continuous loop.

prevSlide() This function moves to the previous slide by decreasing currentSlide by 1. The formula (currentSlide - 1 + slides.length) % slides.length ensures that when we're on the first slide, clicking "previous" takes us to the last slide, keeping the transition circular.

Both functions call showSlide(currentSlide) to update the active slide after the index change.

4. Making the Buttons Interactive & Auto-Transitioning Slides

Let's finish the JS code by making the buttons interactive using addEventListener.

JavaScript 

      
23nextBtn.addEventListener('click', nextSlide);

      
24prevBtn.addEventListener('click', prevSlide);

      
25

      
26setInterval(nextSlide, 20000);

nextBtn.addEventListener('click', nextSlide);, prevBtn.addEventListener('click', prevSlide); These add click event listeners to the "next" and "previous" buttons. Clicking them triggers the nextSlide() and prevSlide() functions, letting users navigate through the slides manually.

setInterval(nextSlide, 20000); This sets up an automatic transition between slides every 20 seconds (20000 milliseconds), creating an auto-sliding effect.

Bringing It All Together

This image slider code seamlessly combines HTML, CSS, and JavaScript to create an interactive experience. With intuitive controls for manual navigation and automatic slide transitions, it offers a flexible foundation for adding dynamic, engaging visual content to any website.

Bonus Touch: Smooth It Out

Elevate your slider with a smooth fade-in effect, adding a sleek transition between slides for a more polished experience.

CSS

26  z-index: 1;

      
27}

This CSS makes a slider with smooth fade animations. When a slide becomes .active, it fades in over 1 second thanks to the transition. Switching slides creates a seamless, elegant transition that enhances the overall user experience.
